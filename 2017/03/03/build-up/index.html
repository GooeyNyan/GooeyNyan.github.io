个人建站历程

(这是建boozinyan.com的时候写的)

胡说八道一堆

喂喂，想啥呢，第一次建站呐，怎么可能会是那些羞羞的事！（欸？似乎可以写些羞羞的内容…

第一次总是带着欣喜与期待，像夏日的波子汽水，流过舌尖的冰凉快感……

唔，说起建站最初的动机，大概是学长的建议，加之开学前两周光看CSS Mastery总感觉缺少了实践，摩拳擦掌间把建站的计划大大提前了（原本打算啃完CSS Mastery，但是实际上还是感觉，哇，遥遥无期啊orzzz）。

所以一拍脑袋……

啊，不好意思，刚晕了过去，拍晕了。

嗯嗯，回到正题

感觉第一次建站还是挺折腾的。域名、主机的购买倒还好说。

傻傻不会域名解析折腾了老半天。一开始搞不清楚几种记录类型和主机记录、莫名其妙的记录值的前世今生来龙去脉，两台主机来来回回设置折腾了好久，解析了老半天解析不到主机上（好吧，是当时傻傻没找到主机分配的别名）。

哇，你知道当时我都低落地决定明天再战了，上床上翻翻主机商的网站，突然看到CNAME解析值的喜悦嘛。大冬天穿着一件短袖就下床打开了电脑，一顿捣鼓之后还是被万恶的联通网址纠错导航带走了。（还好没感冒，不然要被某人…）

第二天在可亲可爱的学长的帮助下，终于是成功看到了主机页面（浏览器缓存的问题）。看到WordPress的可萌可萌的盆栽的时候，那欣慰的感觉难以言表……

之后的半天想着顺https的潮流，为用户的安全性着想，而且主机服务里又提供方便地证书获取，折腾起了https。一开始WP装在subdirectory（本来觉得装在根目录太乱），通过.htaccess也能缩短访问的路径。

为实现全站HTTPS，本身.htaccess的规则理解的缺乏，结果还是在根目录重装了WP。之后本来想捣鼓HSTS，看网上的301重定向方法，然而为何到了我这就一直导致重定向次数过多的问题（本以为是因为之前WP装在子目录导致跳转次数过多的原因，重装了WP）。301重定向导致的这个问题，还是没满足HSTS的要求。不过目前站点的安全性应该还是勉强够用了…吧？

乱七八糟折腾了一堆总算对网站有了一点认识，果然还是须躬行。嘛，看着自己折腾出来的猫窝，还是挺欣慰的 (ゝ∀･)

大概这个博客就是记录学习历程咯，自己遇到的一些坑点的记录，比如前两天遇到的::first-line{text-transform:;}。噢对，还有大家不喜闻乐见的跟某人的某些羞羞的东西。哇，我跟你讲啊，她写影评可好啦，改天把她抓来当苦力，嗯，就是这样~

之后的计划，大概是学习vue.js写自己的样式，顺便看CSS Mastery，毕竟学长也说CSS是前端的一大坑。

啊……

“南京，晚安”
